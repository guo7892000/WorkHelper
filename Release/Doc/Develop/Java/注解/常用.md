### 常用注解
```
 @Autowired	自动装配，当其值为false时，是可选装配。
 @Async	异步方法
 @Bean	Bean组件
 @ConfigurationProperties	引入配置文件中的配置分类
 @Component	定义组件。可以指定名称。
 @Controller	处理http请求。一般作为Rest API，由UI调用。
 @ComponentScan	组件扫描。basePackages属性为扫描的包清单数组，其包名可以是本项目内的，也可以是引用的项目。包名可以是全称或前缀。
 @CrossOrigin	可注释在类或方法上。表示跨域的类或方法。
 @Conditional	
 @DeleteMapping	组合注解，简化Delete请求。
 @EnableAsync	启用异步加载
 @EnableAutoConfiguration	条件性
 @EnableWebMvc	启用MVC
 @EnableWebSecurity	启用Web安全
 @GetMapping	组合注解，简化Get请求。
 @ImportResource	引用资源。
 @PostMapping	组合注解，简化Post请求。
 @PutMapping	组合注解，简化Put请求。
 @PathVariable	"获取url路径中的数据。例如：http://127.0.0.1:8090/gril/test/hello/520"
 @PropertySource	属性源。
 @Profile	配置
 @RestController	Spring4之后新加的注解，原来返回json需要@ResponseBody配合@Controller
 @RequestMapping	配置url映射
 @RequestParam	"获取请求参数的值。例如：http://127.0.0.1:8090/gril/test/say?id=5201。"
 @SpringBootConfiguration	SpringBoot配置注解
 @SpringBootApplication	SpringBoot应用。可以解决根类或者配置类（我自己的说法，就是main所在类）头上注解过多的问题，一个@SpringbootApplication相当于@Configuration,@EnableAutoConfiguration和 @ComponentScan 并具有他们的默认属性值
 @SuppressWarnings	忽略警告
 @Service	业务逻辑接口
 @Scope	singleton为单实例，prototype为每次创建新实体。
 @ServerEndpoint	WebSocket服务器
 @Value	引入配置文件中的配置值
 @Resource	自动装配Bean，类似@Autowired。但不支持@Primary。且不支持@Autowired false（可选装配），建议使用@Autowired。
 @Primary	主要的。如一个接口有多个实现，加上该注解的Bean，那么在使用接口注入Bean时，会注入该Bean。
 @Qualifier	标志使用哪一个Bean，需要传入Bean的ID值。
 @Inject	可自动装配Bean，类似@Autowired，支持@Primary。但需要引用javax.inject，且不支持@Autowired false（可选装配）。不建议使用。
```

### 自定义注解
它类似于新创建一个接口文件，但为了区分，我们需要将它声明为@interface。  
`public @interface NewAnnotation {}`
