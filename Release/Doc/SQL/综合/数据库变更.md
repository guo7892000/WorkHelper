### 数据库变更
背景：因为在软件开发过程中，经常要变更数据库，包括新增或修改表、新增或修改函数、新增或修改存储过程、新增或修改JOB、新增或修改包、
新增或修改序列、新增或修改视图、新增或修改存索引等。而这些对象又会经常变化。如果我们能不断累加这些脚本最好，但这会增加比较多的工作。
为了能使前期简单（当然一些重要的对象得每个自行备份好），后期统一提取。需要增加过程中的一些对象收集，如：
```
变更类型：新增、修改、删除
对象类型：表、存储过程、包、函数、视图、序列、索引、触发器、DBMS_JOB、JOB、DBLink、数据初始化
对象编码：表名、存储过程名、包名、函数名、视图名、固化视图名、序列、索引、触发器名
对象名称：对象的描述
变更描述：主要是针对修改的内容，增加描述。有些存储过程，就不全量发布，直接提取新增的内容，打开对应环境，增加新增的内容。
变更脚本：脚本尽量在最后提取，才最准确。数据初始化，还是建议提供脚本。因为数据是会变化的，无从得知初始化的是哪部分数据。
```
