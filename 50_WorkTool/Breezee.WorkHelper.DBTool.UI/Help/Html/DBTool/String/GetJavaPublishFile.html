获取Java的class文件

一、功能概述

针对修改过的源码，获取其对应的class文件，方便于替换发布。
注：每次修改源码后，记得要重新编译后再获取，否则得到的class文件可能不是最新的！！

三、操作步骤

1、【代码目录】：输入或选择要源码所在的目录，即src的上一级目录！
2、【class目录】：编译后的项目最新class目录。注：如项目是JSP那种包括页面的项目，并且使用JBOSS编译，那么其路径需要指定到JBOSS编译的War包下，如D:\jboss-as-7.1.1\standalone\deployments\Test.war。
如果是不包含页面的项目，可以直接使用项目路径下的Target目录，如：D:\TestInf\target\classes。
3、【复制到目录】：将跟代码变更相关的class文件，复制到指定的目录。
4、【修改时间】：我们可以选择开始和结束时间，然后点击【获取代码变更清单】，会将这段时间内修改的代码文件加入到【变化文件录入】网格中。
另外，我们也支持从Excel中复制一列数据，然后按crtl + v键（或右击网格，再点粘贴），可以把数据加到网格中。
5、【覆盖类型】：包括！
覆盖：任何日期的复制，文件都会直接复制到【复制到目录】下，并覆盖之前日期复制的文件。
覆盖当天：在【复制到目录】下创建一个当天日期目录，然后复制的文件放该目录下。当天多次复制，都会使用该目录，并覆盖之前的文件。
每次新增：每次生成，都在【复制到目录】下创建一个当天日期-时分秒目录，文件放入其中。当天多次复制，每次都放入不同目录。
6、排除目录或文件选项：
排除目录：排除相对于【代码目录】的相对路径为【排除目录】文本框内输入的子目录。
排除文件：排除相对于【代码目录】的相对路径为【排除文件】文本框内输入的文件。
7、【配置名称】：必填项，可以保存复制所需配置项，以方便下次使用。
8、点击【复制文件】：将会异步复制文件，并保存配置。
复制成功后，会在【生成结果】中显示本次复制的文件清单。

四、逻辑与业务规则

注意点：支持逗号（中英文）、分号（中英文）、冒号（中文）、竖线（英文）分隔的多个排除项配置！另外：会过滤以.开头的文件或文件夹。
