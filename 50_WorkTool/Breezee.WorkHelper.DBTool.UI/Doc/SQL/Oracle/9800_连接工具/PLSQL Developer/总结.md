## 总结
使用过程中的技巧或问题解决总结。
### 生成SELECT、INSERT等SQL
拖动表到SQL输入中，会弹出生成SQL类型；然后选择INSERT，即可生成新增SQL。还有其他项：Name、Select Into、Select Loop、
Select、Insert、Update、Delete、Description、Properties、DDL、Query Builder。
### PL/SQL Developer使用instantclient的配置
一. 目录结构  
```
    D:\install\PLSQL
        |-- instantclient_11_2
            |-- tnsnames.ora
        |-- PLSQL Developer
        |-- readme.txt
```
二. 环境变量  
```
    NLS_LANG = SIMPLIFIED CHINESE_CHINA.ZHS16GBK
    TNS_ADMIN = D:\install\PLSQL\instantclient_11_2
```
三. PL/SQL Developer 环境设置  
 1. 位置:工具 -> 首选项 -> 连接  
 2. 设置  
```
        Oracle主目录名  = D:\install\PLSQL\instantclient_11_2
        OCI库 = D:\install\PLSQL\instantclient_11_2\oci.dll 
```
tnsnames.ora文件：  
```
172.17.x.xxx =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (ADDRESS = (PROTOCOL = TCP)(HOST = 172.17.x.xxx)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = orcl)
    )
  )
```
### 修改包或存储过程后查看显示乱码问题的解决方案
1.安装PLSQL Developer 16版本。注：16版本以下有环境变量设置；其他版本只能在Windows中的环境变量中增加设置。     
2.打开PLSQL Developer，进入工具 -> 首选项 -> 连接，然后在环境变量中增加NLS_LANG = SIMPLIFIED CHINESE_CHINA.ZHS16GBK。  
3.重启PLSQL Developer即可。  

### 针对包声明和包体，可以同时保存为一个文件。
右击包体，然后点击【编辑说明和体(S)】,即可以同时打开说明和体。上方左边为说明，上方右边为体，然后保存时将说明和体都保存在一个文件中。  
如果我们是右击包体，然后点击【编辑(Q)】，那么就只打开体，保存的文件也只是体；操作也同样适用用说明。

### 针对包体或存储过程一执行就没响应问题
原因通常是其他人或本人上一次执行该包体或存储过程时，没有正常结束。我们要找出那个Session，结束掉。具体操作：  
PLSQL Developer > Tools > Sessions > 查看所有Sessions > 选中执行内容为对应包体或存储过程的那条，右击选择【Kill】。
