## JBoss
### IDEA中使用JBoss配置
构建、执行、部署 > 应用程序服务器 > +  > JBoss/WildFly 7.1.1.Final。

### 将jar包转换为文件夹方式部署
前提：有一个web应用，如PAStore.jar，部署在【/opt/app/appSrv/jboss-as-7.2.0.Final/standalone/deployments/】中：  
命令行方式	
```
cd /opt/app/appSrv/jboss-as-7.2.0.Final/standalone/deployments/								//进入发布目录
mkdir PAStore.war1									//创建目录PAStore.war1
unzip PAStore.war -d ./PAStore.war1/				//解压PAStore.war到PAStore.war1目录
rm -f PAStore.war									//删除压缩包PAStore.war
mv  PAStore.war1 PAStore.war						//得命名目录PAStore.war1为PAStore.war
touch PAStore.war.dodeploy							//创建文件PAStore.war.dodeploy
//执行重新发布命令
ps -ef|grep jboss						//显示jboss进程
kill -9 98827							//杀掉进程
nohup ./standalone.sh -b 0.0.0.0 &		//启动jboss进程
tail -f nohup.out						//显示日志
```

### ps -ef|grep jboss显示的具体内容
```
UID：表示用户ID
PID：表示进程ID
PPID：表示父进程号
C：表示CPU的占用率
STIME：进程的启动时间
TTY：登入者的终端机位置
TIME：表示进程执行起到现在总的CPU占用时间
CMD：表示启动这个进程的命令
```
